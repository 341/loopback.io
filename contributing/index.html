Contributing to LoopBack

LoopBack has an active and growing open-source community. All work on the project occurs in GitHub; mostly in the loopback repository, but also in:

  • loopback-datasource-juggler
  • loopback-boot
  • loopback-phase
  • loopback-boot
  • generator-loopback

See LoopBack documentation for a complete list.

Contributing code

If you're interested in contributing, be sure to read the code contribution guidelines first. Here's a list of open issues that's a good place to start.

Contributing to documentation

We welcome input and help with the documentation. If you're interested in contributing to the documentation, see Contributing to LoopBack documentation.

Reporting issues

You can also contribute by reporting new issues; please see Reporting issues for general guidelines. New issues start in an un-labelled state. During triage, if the requirements and/or steps-to-reproduce are clear, the issue is labelled as "bug" or "enhancement". If the issue description is not good enough, leave a comment asking the reporter to provide the specific information needed, and label the issue as "triage". Issues that are questions will be redirected to the mailing list and closed straight away.
